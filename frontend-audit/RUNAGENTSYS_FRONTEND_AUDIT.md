# RunAgentSys Frontend Audit Checklist

## Positioning to verify

The frontend should sell RAS as:

- Managed AI Agent Service.
- Full setup on customer's VPS/cloud.
- RAS1 + RAS2 package.
- Secure OAuth/API connection layer.
- Social/CSKH/CRM/content/video workflow system.

It should not look like:

- VPS rental only.
- Source-code sale only.
- Generic chatbot only.
- Old Starter/Growth/Agency SaaS pricing unless intentionally revived.

## Required sections

### Hero

Should clearly say:

> Deploy a managed AI agent system for customer care, social operations, leads, and content workflows.

Mention:
- RAS1 Social/CSKH/CRM Agent
- RAS2 Content/Video Workflow Agent

### Architecture / What you get

Must show:
- VPS/cloud setup
- Hermes/RAS1
- OpenClaw/RAS2
- Zernio secure OAuth/secrets
- gog CLI / Google Workspace
- iSocial multi-platform workflows
- chatbot CSKH template

### Quick Contact Widget

Should remain prominent:
- Quick Contact
- Telegram = Customer Care Assistant
- WhatsApp/LINE = Customer Care Team

Known Telegram link:
- `https://t.me/Runagentsysbot`

### Pricing

Recommended wording:

> Custom RAS Deployment starts from $100/month. Final pricing depends on number of channels/accounts, workflow scope, AI usage, VPS/cloud requirements, and support level. Setup fee may apply.

Better if frontend explains setup fee + monthly fee.

### Trust / Security

Mention:
- OAuth/API where possible
- no password sharing when official OAuth/API is available
- secrets handled securely
- revoke permissions when needed
- customer-owned VPS/cloud option

### Legal / Contact

Needs real data before publishing:
- legal entity/company info if available
- contact channels
- refund/cancel/SLA wording
- privacy/data handling note

Do not invent legal entity or guarantees.

## Current known frontend concern

User previously corrected that quick contact buttons are customer-facing chat redirect buttons, not technical bot options. Frontend should keep this customer-facing.
